**<u>Propuesta de Proyecto</u>**

1. <span id="_Toc52661346" class="anchor"></span>**Datos Generales**

    - **Título del Proyecto:** Plataforma de Gestión y Análisis de Herramientas de IA para Estudiantes de Ingeniería de Sistemas.
    - **Área:** Ingeniería de Software / Inteligencia de Negocios.
    - **Duración Estimada:** 4 meses.
    - **Ubicación:** Universidad Privada de Tacna (Campus Virtual).

2. <span id="_Toc52661347" class="anchor"></span>**Planteamiento del Problema**

    Los estudiantes de ingeniería de sistemas se enfrentan a una oferta abrumadora de herramientas de Inteligencia Artificial (Copilots, LLMs, Generadores de Código). No existe una guía centralizada que valide su utilidad académica, costos y curva de aprendizaje. Esto resulta en el uso ineficiente de tecnologías o la adopción de herramientas que no cumplen con los estándares académicos, afectando la calidad de los proyectos formativos.

3. <span id="_Toc52661348" class="anchor"></span>**Objetivos**

    3.1. Objetivo General
    
    Implementar una plataforma tecnológica que permita catalogar, evaluar y analizar herramientas de IA, proporcionando a los estudiantes de la EPIS información basada en datos para su toma de decisiones académicas.

    3.2. Objetivos Específicos
    
    - Diseñar una base de datos escalable para almacenar información de herramientas y reseñas.
    - Automatizar la recolección de métricas de uso simuladas para validar modelos de análisis.
    - Desplegar un dashboard de inteligencia de negocios accesible vía web.

4. <span id="_Toc52661349" class="anchor"></span>**Alcance**

    El proyecto incluye:
    - **Backend:** Base de datos en Google Cloud SQL.
    - **Infrastructure as Code:** Scripts de Terraform.
    - **Data Processing:** Scripts de Python para ETL y simulación.
    - **Frontend/BI:** Dashboard en Looker Studio.
    
    El proyecto **no** incluye:
    - Desarrollo de una aplicación móvil nativa.
    - Venta o reventa de licencias de software.

5. <span id="_Toc52661350" class="anchor"></span>**Metodología**

    Se propone el uso de **Scrum** para el desarrollo del software, con sprints de 2 semanas. Para la gestión de la infraestructura, se seguirán prácticas de **DevOps** (Infraestructura como Código, Control de Versiones).

6. <span id="_Toc52661351" class="anchor"></span>**Cronograma Tentativo**

    | Mes | Actividad Principal | Entregable |
    | :--- | :--- | :--- |
    | Mes 1 | Análisis y Diseño | Documentos FD01, FD02, FD03 |
    | Mes 2 | Arquitectura e Infraestructura | Terraform Scripts, Instancia BD |
    | Mes 3 | Desarrollo e Integración | Scripts Python, Datos de Prueba |
    | Mes 4 | Pruebas y Despliegue | Dashboard Final, Informe FD05 |
